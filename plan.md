# 開発計画 (plan.md)

本ドキュメントは、アプリケーション仕様書（`specs.md`）に基づき、認証およびユーザー管理機能を持つサンプルアプリケーションの開発ステップを定義したものです。

## Step 1: Hello Worldのソースとテストの作成
基盤となるWebサーバーと、シンプルな画面を表示する初期構築を行います。

* **タスク内容**:
  * Go言語標準の `net/http` および `html/template` を用いたWebサーバーの立ち上げ。
  * `Hello World` を返すエンドポイントの作成と、対応するテンプレートの作成。
  * Tailwind CSS（CDNまたはビルドプロセス）の導入と初期設定。
  * HTMX および Alpine.js をHTMLテンプレートに組み込む準備。
  * 上記のルーティングおよびレスポンスが正常に機能することを確認するための自動テスト（Goの `testing` パッケージを利用）の作成。

## Step 2: ビルド＆デプロイ
作成したアプリケーションをAWS Lambdaおよびオンプレミス環境で動作させるためのビルド・デプロイ環境を整備します。

* **タスク内容**:
  * クラウド（AWS Lambda）での稼働を前提としたビルドスクリプト・設定（例：AWS SAM、Serverless Framework等、またはカスタムスクリプト）の作成。
  * API Gateway と Lambda を統合し、HTTPリクエストを処理する仕組みの構築（Goの `aws-lambda-go` 等の利用）。
  * オンプレミス環境（Windowsの `nssm` など）でも同一のバイナリが動作するよう、環境変数で起動モード（Lambda vs ローカルサーバー）を切り替えるアーキテクチャの導入。
  * CI/CDパイプラインまたは手動デプロイ手順のドキュメント化。

## Step 3: ユーザー認証（SQLite）、パスワード更新
ローカルデータストアであるSQLiteを用いたアプリケーション内部のユーザー管理と機能実装を行います。

* **タスク内容**:
  * SQLiteデータベースへの接続と初期スキーマ設定（ユーザーテーブル等の作成）。
  * アプリケーション内部のプロファイル・権限情報を管理するCRUD処理の実装。
  * ログイン後画面の作成と、ユーザー自身がパスワード更新（DB上の更新と、後続のStep 4に向けた準備）を行えるフォームの実装（HTMXによる非同期送信、Alpine.jsによるUI操作）。
  * データベースのバックアップとリストアの仕組み構築（S3への定期アップロード処理、およびS3からのダウンロード処理）。

## Step 4: Cognitoとの連携
AWS Cognitoをメインの認証プロバイダとして統合し、堅牢な認証システムを完成させます。

* **タスク内容**:
  * AWS Cognito ユーザープールの構築と初期設定。
  * サインアップ、サインイン（ログイン）、ログアウト処理のCognito APIとの連携実装。
  * Cognitoで認証されたユーザーのセッショントークン（JWTなど）の検証と、SQLite内のユーザー情報との同期・連携処理。
  * Step 3 で作成したパスワード更新機能を、Cognitoのパスワード変更APIを呼び出すように改修。
  * HTMXとAlpine.jsを活用した、シームレスな認証UI（ローディング表示やエラーメッセージの表示）の完成。
