# アプリケーション仕様書

## 1. 概要
本システムは、ログインおよびパスワード変更機能の実装例を示すサンプルアプリケーションである。
AWS環境（Lambda等）とオンプレミス環境の両方で動作可能な柔軟なデプロイ構成を持ち、認証とユーザー管理のベースラインを提供する。

## 2. 技術スタック
### 2.1. バックエンド・インフラ
- **言語**: Go
- **データベース**: SQLite
- **ストレージ**: Amazon S3
  - 用途: SQLiteデータベースファイルのバックアップおよびリストア
- **実行環境**: AWS Lambda （デフォルト）
  - ※設定変更のみでオンプレミス環境（nssm等を用いたWindowsサービス化など）でも稼働可能なポータビリティを持たせる。

### 2.2. フロントエンド
- **HTML生成**: Go標準 `html/template`
- **動的UI/通信**: HTMX（サーバーとの非同期通信用）
- **UI制御**: Alpine.js（ブラウザ内での軽量な状態管理・動作制御用）
- **デザイン**: Tailwind CSS

### 2.3. 認証・ユーザー管理
- **メイン認証**: Amazon Cognito
- **内部データ管理**: SQLite
  - アプリケーション内部でのユーザープロファイルや状態管理に利用。Cognitoと連携して動作する。

## 3. システム構成と機能要件

### 3.1. 認証およびユーザー管理機能
- **ログイン機能**: Cognitoを利用したセキュアなユーザー認証を行う。
- **パスワード更新機能**: ログイン後のユーザー自身によるパスワード変更を可能とする。
- **データ連携**: Cognito側での認証情報管理と並行し、SQLite側でアプリケーション固有のユーザー情報（権限など）を管理する。

### 3.2. データ永続化と可用性
- SQLiteをローカルのデータストアとして利用する。
- データの永続性と障害復旧（ディザスタリカバリ）を担保するため、SQLiteのデータベースファイルを定期的にS3へバックアップし、必要時にS3からリストアできる仕組みを構築する。

### 3.3. デプロイメント・アーキテクチャ
- Goのシングルバイナリの特性を活かし、実行環境への依存を最小限に抑える。
- クラウド環境（AWS Lambda）でのサーバーレス稼働を前提としつつ、環境変数や設定ファイルの切り替えのみで、オンプレミス環境（例: nssmによるWindowsサービス起動）でも同一のバイナリがシームレスに動作する設計とする。

### 3.4. ユーザーインターフェース (UI/UX)
- Goの `html/template` によるサーバーサイドレンダリング(SSR)をベースとする。
- HTMXを活用してページ全体の再読み込みを減らし、SPA（Single Page Application）ライクなスムーズな画面遷移と操作性を実現する。
- Alpine.jsを組み合わせて、ドロップダウンやトグル、モーダル表示などのフロントエンドにおける細かな振る舞いをシンプルに実装する。
- Tailwind CSSにより、カスタマイズ性が高く一貫したモダンなデザインを適用する。
